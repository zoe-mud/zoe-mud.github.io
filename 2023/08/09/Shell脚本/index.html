Shell脚本

发表于2023-08-09|更新于2023-08-11
|阅读量:

shell脚本这个概念我知道大学毕业才有点概念，接下来我将写出我的学习体会。

shell分很多种，我主要使用Linux，所以自带的bash。

打开终端，屏幕上出现提示符：用户名@主机名

The art of command lines[手册](“the-art-of-command-line/README-zh.md at master · jlevy/the-art-of-command-line (github.com)“)

基础命令

熟悉至少一个基于文本的编辑器（vim）。

#打开文件
$ vim filename
#打开insert模式
$ i
#回到命令模式
按esc
#保存当前文件并退出Vim
$:wq
#不保存文件并退出Vim
$:q!
#删除当前行
$ dd

学会如何使用 man 命令去阅读文档。学会使用 apropos 去查找文档。知道有些命令并不对应可执行文件，而是在 Bash 内置好的，此时可以使用 help 和 help -d 命令获取帮助信息。你可以用 type 命令 来判断这个命令到底是可执行文件、shell 内置命令还是别名。

#print work dirctionary
$ pwd
#read documents
$ man
#search the keywords in the page of man_docs
$apropos date
# get the bash internal command and help_info（internal command = builtin）
$ help cd
#get the help_info about alias
$ help -d
#Determine whether a command is an executable, shell-builtin, or alias
$type -a ls

学会使用 > 和 < 来重定向输出和输入，学会使用 | 来重定向管道。明白 > 会覆盖了输出文件而 >> 是在文件末添加。了解标准输出 stdout 和标准错误 stderr。了解其对应的接口：0,1,2。

#使用>将命令的输出写入文件。如果文件已经存在，则会覆盖文件内容。
$ command > file.txt
#使用>>将命令的输出添加到文件的末尾，而不会覆盖原有内容
$ command >> file.txt
#使用<将文件中的内容作为命令的输入。
$ command < file.txt
#使用|将一个命令的输出作为另一个命令的输入，command1 会生成输出，然后通过管道将其传递给 command2
$ command1 | command2
#例如：过滤数据，将打开名为 file.txt 的文件，并将其内容传递给 grep 命令进行过滤。grep 命令将会搜索包含关键词 keyword 的行并输出。
$cat file.txt | grep keyword

#标准输出（stdout）：命令的正常输出都会发送到stdout，默认情况下是在终端上显示。可以使用输出重定向符号>或>>将stdout重定向到文件，从而将输出保存到文件中。
#标准错误（stderr）：命令的错误输出会发送到stderr，默认情况下也是在终端上显示。可以使用重定向符号2>将stderr重定向到文件，即将错误信息保存到文件中。
#使用>重定向stdout，并使用2>重定向stderr到不同的文件。
$ command > output.txt 2> error.txt
#使用&>将stdout和stderr都重定向到同一个文件。
$ command &> output_and_error.txt

#进入目录/proc/pid/fd，可以查看口情况
$
#ps命令：查看当前运行程序
#a：显示当前终端下的所有进程信息，包括其他用户的进程。
#u：使用以用户为主的格式输出进程信息。
#x：显示当前用户在所有终端下的进程。
$ps aux
#查看cup、内存情况
$top
#以树状图的方式展现进程之间的派生关系
$ pstree -aup
#调到前台
$ fg num

一、程序

命令由程序名+参数组成。如以下

#打印当前时间
$ date 
#打印参数
$ echo hello
#参数中有空格
$ echo "hello word"
#下述方法也行
$ echo hello\ world

程序执行是系统先在路径中搜索，以下是系统设置的路径

$ echo $PATH

使用which获取程序的路径

$ which echo
/usr/bin/echo

但是获取不到shell的built-in的路径，如：cd、exec、return，因为cd is not the executable program as ls, but it is the internal command related to shell data and control. 

$ which cd
...
$type -a cd
cd is a shell builtin

变量与参数

#定义一个变量
$foo=var
#使用变量
$$(foo)
#在字符串中使用变量
"we are in $(pwd)”
"the time is $(date)"
"the content is $(foo)"
#常见参数
$0 #脚本名
$1 #第一个参数
$# #参数数量
$@ #所有参数 
$$ #命令进程ID
$? 未知的

查找文件

#ls
$ ls foo/foo
#find
$ find . -name "*.tmp" -type f
$ find . -path '**/test/*.py' -type f
#查找一天以内修改过的时间
$ find . -mtime -1
#. 在此表示当前目录;-name “*ab” 表示查找以ab结尾的文件或文件名;-exec表示执行什么命令。后面跟要执行的命令。此处是rm -f，表示不确认删除;{} \;表示把查找到的结果发送到此来。
$ find . -name "*ab*" -exec rm -f {} \;
#在{}和\之间必须要有空格，否则会报上面的错

查找文件内容

#grep:过滤/查找出指定内容
$grep fopobar foobar.sh
#-R表示遍历整个目录
$grep -R foobar foobar.sh 
#fzf:交互式查找
$ cat a.c | fzf

目录列表

#递归查看所有
$ls -R
#树形结构
$tree
